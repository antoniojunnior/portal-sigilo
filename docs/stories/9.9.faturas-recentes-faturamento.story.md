## Story

**As a** admin logado,
**I want** ver as faturas recentes da minha assinatura em `/configuracoes/faturamento`,
**so that** eu possa acompanhar o histórico de pagamentos sem acessar o Asaas.

## Acceptance Criteria

### AC1 — Novo Route Handler `GET /api/billing/invoices`

- Verifica sessão e `role === "admin"`.
- Lê `asaas_customer_id` da org.
- Se ausente → retorna `{ invoices: [] }`.
- Se presente → chama `GET ${ASAAS_BASE_URL}/v3/payments?customer={id}&limit=5&sort=dateCreated&order=desc`.
- Retorna lista normalizada:

```ts
interface Invoice {
  id: string;
  valor: number;
  vencimento: string;     // ISO date
  status: "RECEIVED" | "PENDING" | "OVERDUE" | "CANCELLED";
  descricao: string | null;
  invoice_url: string | null;   // link para boleto/fatura Asaas (campo invoiceUrl)
}
```

### AC2 — UI: seção "Faturas Recentes" em `/configuracoes/faturamento`

- Nova seção abaixo de "Assinatura Ativa".
- Tabela com colunas: Vencimento | Descrição | Valor | Status | Ação.
- Badge de status com cores:
  - RECEIVED → verde (pago)
  - PENDING → amarelo (aguardando)
  - OVERDUE → vermelho (atrasado)
  - CANCELLED → cinza (cancelado)
- Coluna "Ação": link "Ver fatura" abre `invoice_url` em nova aba (se disponível).
- `invoices.length === 0` → mensagem "Nenhuma fatura registrada.".

### AC3 — Link "Ver histórico de faturas" em /configuracoes

- Após story 9.7 remover o botão estático, este story não adiciona novo CTA em /configuracoes.
- A seção de faturas fica apenas em `/configuracoes/faturamento`.

### AC4 — Loading e erro padrão

- Skeleton tabela (3 linhas) durante fetch.
- Erro → `AlertTriangle` com mensagem.
