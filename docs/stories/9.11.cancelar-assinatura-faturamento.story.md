## Story

**As a** admin logado com assinatura ativa,
**I want** cancelar minha assinatura dentro de `/configuracoes/faturamento` com confirmação explícita,
**so that** eu encerre a assinatura com ciência das consequências e o sistema reflita o novo estado.

## Acceptance Criteria

### AC1 — Novo Route Handler `DELETE /api/billing/cancel`

- Verifica sessão e `role === "admin"`.
- Lê `asaas_customer_id` da org no Firestore.
- Obtém `subscription_id` via `GET /v3/subscriptions?customer={id}&status=ACTIVE&limit=1` (server-side, não confia em valor do body cliente).
- Chama `DELETE ${ASAAS_BASE_URL}/v3/subscriptions/{subscription_id}`.
- Se Asaas retornar sucesso:
  - Atualiza Firestore: `orgs/{orgId}.plano_ativo = "cancelado"`.
  - Grava audit log: `{ acao: "assinatura_cancelada", detalhes: { subscription_id } }`.
- Retorna `{ ok: true }`.
- Erro Asaas → 502 com mensagem.

### AC2 — UI: botão "Cancelar Assinatura" (Zona de Perigo)

- Nova subseção "Cancelamento" em `/configuracoes/faturamento`, abaixo das faturas.
- Estilo danger (borda vermelha, fundo danger-surface) — padrão do projeto.
- Botão "Cancelar Assinatura" visível apenas quando `subscription_id !== null`.

### AC3 — Modal de confirmação

- Ao clicar no botão → abre Modal com:
  - Título: "Cancelar Assinatura"
  - Corpo: aviso sobre perda de acesso às funcionalidades do plano.
  - Campo de texto: admin deve digitar exatamente `CANCELAR` para habilitar o botão de confirmação.
  - Botão "Confirmar Cancelamento" (danger, desabilitado até digitação correta).
  - Botão "Manter Assinatura" (ghost, fecha modal).

### AC4 — Pós-cancelamento

- Chamada `DELETE /api/billing/cancel` com loading no botão.
- Sucesso → modal fecha, UI atualiza: badge "CANCELADO", botões "Alterar Plano" e "Cancelar Assinatura" somem.
- Erro → mensagem dentro do modal, modal permanece aberto.

### AC5 — Audit log imutável

- Documento gravado em `audit_logs` NÃO pode ser atualizado/deletado (regra existente no projeto).
