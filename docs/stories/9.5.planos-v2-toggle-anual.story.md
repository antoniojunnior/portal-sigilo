## Story

**As a** visitante da página `/planos`,
**I want** visualizar os preços em modo mensal ou anual com toggle interativo e copy orientado a benefícios,
**so that** eu tome uma decisão de compra informada — entendendo claramente a economia do plano anual — e o checkout envie o ciclo correto ao Asaas.

## Acceptance Criteria

### AC1 — Tipo `BillingCycle` e extensões de dados

- `BillingCycle = "mensal" | "anual"` adicionado em `src/lib/types/index.ts`
- Interface `PlanoConfig` em `src/lib/types/index.ts` ganha campos opcionais:
  - `economiaAnual?: number`
  - `tagline?: string`
- `src/lib/planos.ts` atualizado com valores:

| Plano | precoMensal | precoAnual | economiaAnual | tagline |
|---|---|---|---|---|
| Entrada | 117 | 97 | 240 | "Para empresas que estão começando" |
| Gestão | 227 | 197 | 360 | "Para quem precisa de controle total" |
| Enterprise | null | null | null | "Para grupos e organizações complexas" |

### AC2 — Helper `createPaymentLink` extraído

- `src/lib/asaas/createPaymentLink.ts` criado com a lógica de criação de payment link Asaas extraída de `src/app/api/checkout/create/route.ts`
- Assinatura: `createPaymentLink(plano: PlanoId, ciclo: BillingCycle): Promise<{ url: string }>`
- Tabela de preços por ciclo dentro do helper:

```ts
const PLANOS_CONFIG = {
  entrada: {
    mensal: { nome: "Portal Sigilo — Entrada (Mensal)", value: 117.00, cycle: "MONTHLY" },
    anual:  { nome: "Portal Sigilo — Entrada (Anual)",  value: 97.00,  cycle: "YEARLY" },
  },
  gestao: {
    mensal: { nome: "Portal Sigilo — Gestão (Mensal)", value: 227.00, cycle: "MONTHLY" },
    anual:  { nome: "Portal Sigilo — Gestão (Anual)",  value: 197.00, cycle: "YEARLY" },
  },
} as const;
```

- `src/app/api/checkout/create/route.ts` refatorado para importar `createPaymentLink` do helper (comportamento inalterado para chamadas sem `ciclo` — default `"mensal"`)

### AC3 — Componente `BillingToggle`

- Arquivo criado: `src/app/planos/BillingToggle.tsx`
- Interface:
  ```tsx
  interface BillingToggleProps {
    value: BillingCycle;
    onChange: (ciclo: BillingCycle) => void;
  }
  ```
- Toggle pill com dois botões: "Mensal" e "Anual"
- Estado padrão: `"anual"` (controlado pelo pai)
- Badge inline `"2 meses grátis"` visível apenas quando ciclo = anual
- Tokens de badge: `bg-[var(--color-success-surface)] border border-[var(--color-success)]/30 text-[var(--color-success)]`
- **Acessibilidade (emenda A-1 + S-1):**
  - Wrapper com `role="group"` e `aria-label="Período de cobrança"`
  - Wrapper com `min-h-[44px]` (WCAG 2.5.5 touch target)
  - `aria-pressed={value === "mensal"}` e `aria-pressed={value === "anual"}` nos botões
  - `onKeyDown` no wrapper: ArrowLeft → "mensal", ArrowRight → "anual"
- **Proibido:** uso de `cn()` — usar `[...].filter(Boolean).join(" ")`

### AC4 — `PlanoCard` v2 (extensão da interface existente)

- `src/app/planos/PlanoCard.tsx` atualizado:
  - Interface `PlanoCardProps` ganha:
    - `ciclo: BillingCycle`
    - `isCurrentPlan?: boolean`
    - `onAction?: () => void`
  - Preço exibido conforme ciclo: `ciclo === "anual" ? plano.precoAnual : plano.precoMensal`
  - Bloco de precificação por ciclo:
    - Anual: preço principal (--text-3xl) + total anual cobrado + badge economia (`bg-[var(--color-success-surface)]`) + preço mensal riscado como comparador
    - Mensal: preço principal + linha "ou R$ X/mês no plano anual"
  - **Token de tipografia (emenda B-1):** preço principal usa `text-[var(--text-3xl)]`, não `text-[var(--text-4xl)]`
  - **Peso (emenda B-3):** usar `font-bold`, não `font-extrabold`
  - CTAs diferenciados por ciclo e plano:

| Plano | Ciclo | CTA |
|---|---|---|
| Entrada | mensal | "Começar agora" |
| Entrada | anual | "Garantir desconto anual" |
| Gestão (destaque) | mensal | "Escolher o mais popular" |
| Gestão (destaque) | anual | "Garantir desconto anual" |
| Enterprise | — | "Falar com vendas" |

  - Card destaque em ≥ md: `md:-translate-y-1 md:shadow-[var(--shadow-lg)]` (emenda A-5 — sem `scale-[1.03]`)
  - `tagline` exibida abaixo do nome do plano quando presente
  - **Proibido:** uso de `cn()` — usar `[...].filter(Boolean).join(" ")`

### AC5 — Page `/planos` v2

- `src/app/planos/page.tsx` convertido para `"use client"`
- Estado `const [ciclo, setCiclo] = useState<BillingCycle>("anual")` — default anual
- Layout atualizado com:
  - Trust signal pré-decisão acima do fold: `"LGPD · ISO 27001 · Criptografia ponta a ponta · Suporte incluso"`
  - H1: `"Proteção real. Conformidade garantida."` — token `text-[var(--text-hero)]`, peso `font-bold` (emendas B-1, B-3)
  - Subtítulo: `"Implante um canal de denúncias LGPD-compliant em minutos — sem jurídico, sem infraestrutura."`
  - `<BillingToggle value={ciclo} onChange={setCiclo} />` centralizado
  - Grid de `<PlanoCard plano={plano} ciclo={ciclo} />` para cada plano
  - Rodapé: `"Sem fidelidade no plano mensal. Cancele quando quiser."`
- `data-portal` no `<main>` mantido (light-forced)

### AC6 — API `POST /api/checkout/create` com suporte a ciclo

- Body expandido: `{ plano: "entrada" | "gestao", ciclo?: "mensal" | "anual" }`
- Ciclo padrão (backward-compatible): `"mensal"` se não informado
- Validação server-side: `ciclo` deve ser `"mensal"` ou `"anual"` se presente
- Usa `createPaymentLink(plano, ciclo)` do helper `src/lib/asaas/createPaymentLink.ts`
- Enterprise não gera chamada ao endpoint (CTA aponta para vendas)

### AC7 — Qualidade

- `npx tsc --noEmit` → EXIT:0
- `npm run lint` → zero erros novos
- Zero uso de `cn()` nos arquivos modificados
- Zero `text-[var(--text-4xl)]` nos arquivos modificados
- Zero `font-extrabold` nos arquivos modificados
- `bg-[var(--color-success)]/10` não presente — substituído por `bg-[var(--color-success-surface)]`
