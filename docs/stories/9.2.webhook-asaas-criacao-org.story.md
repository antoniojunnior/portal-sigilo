## Story

**As a** novo cliente que acabou de pagar pelo Portal Sigilo via Asaas,
**I want** que minha organização seja provisionada automaticamente com usuário admin e e-mail de boas-vindas,
**so that** eu possa acessar o dashboard em minutos sem precisar de intervenção manual da equipe do Portal Sigilo.

## Acceptance Criteria

1. **Webhook endpoint seguro**
   - Firebase Function `webhookAsaas` exposta em `/webhook/asaas` (HTTP POST)
   - Valida header `asaas-access-token` contra `ASAAS_WEBHOOK_TOKEN` antes de processar qualquer payload
   - Retorna 401 imediatamente se token inválido ou ausente
   - Retorna 200 mesmo em eventos ignorados (idempotência e conformidade com Asaas retry policy)

2. **Evento `PAYMENT_CONFIRMED` / `PAYMENT_RECEIVED`**
   - Verifica se org já existe (campo `asaas_customer_id` em `orgs`) — idempotência
   - Se não existe: cria documento em `orgs` com campos:
     `id, nome, slug (gerado do nome), plano_ativo, asaas_customer_id, asaas_subscription_id, data_inicio, data_renovacao, criado_em, configuracoes (defaults)`
   - Cria usuário Firebase Auth (email + senha temporária) para o admin
   - Cria documento em `users` com `role: admin, org_id, email, ativo: true, criado_em`
   - Envia e-mail de boas-vindas via coleção `mail` (Firebase Trigger Email Extension)
   - Grava `audit_log` com `acao: 'org_created', org_id, detalhes: { plano, asaas_customer_id }`

3. **Evento `PAYMENT_OVERDUE`**
   - Atualiza `orgs/{org_id}.plano_ativo` → `'suspenso'`
   - Grava `audit_log` com `acao: 'plan_suspended', org_id`

4. **Evento `SUBSCRIPTION_CANCELED` / `SUBSCRIPTION_INACTIVATED`**
   - Atualiza `orgs/{org_id}.plano_ativo` → `'cancelado'`
   - Grava `audit_log` com `acao: 'plan_canceled', org_id`

5. **Segurança e audit**
   - `ASAAS_WEBHOOK_TOKEN` presente APENAS em variável de ambiente das Firebase Functions (nunca no código)
   - Nenhum dado de cartão ou dados financeiros armazenado no Firestore
   - Todo evento processado com sucesso gera `audit_log` com `timestamp` server-side
   - Audit logs seguem regra de imutabilidade (Firestore Rules bloqueiam update/delete)

6. **E-mail de boas-vindas**
   - Template com: nome da org, plano contratado, link de acesso ao dashboard, senha temporária
   - Enviado via coleção `mail` (padrão Firebase Trigger Email já usado na Story 9.2-sched de relatórios)
   - Em caso de falha no e-mail: org ainda é criada, falha é logada mas NÃO impede o provisionamento

### Modelo de Functions v2 (padrão do projeto)

```typescript
import { onRequest } from "firebase-functions/v2/https";
import { setGlobalOptions } from "firebase-functions/v2";

setGlobalOptions({ maxInstances: 10, region: "southamerica-east1" });

export const webhookAsaas = onRequest(async (req, res) => {
  // validação de token
  const token = req.headers['asaas-access-token'];
  if (token !== process.env.ASAAS_WEBHOOK_TOKEN) {
    res.status(401).json({ error: 'Unauthorized' });
    return;
  }
  // processamento
});
```

### Estrutura do payload Asaas (webhook)

```typescript
interface AsaasWebhookPayload {
  event: 'PAYMENT_CONFIRMED' | 'PAYMENT_RECEIVED' | 'PAYMENT_OVERDUE'
        | 'SUBSCRIPTION_CANCELED' | 'SUBSCRIPTION_INACTIVATED';
  payment?: {
    id: string;
    customer: string;          // asaas_customer_id
    subscription?: string;     // asaas_subscription_id
    value: number;
    billingType: string;
    description?: string;
  };
  subscription?: {
    id: string;
    customer: string;
    status: string;
    cycle: 'MONTHLY' | 'YEARLY';
  };
}
```

interface Org {

### Mapeamento plano por valor/ciclo Asaas

```typescript
function determinarPlano(payload: AsaasWebhookPayload): string {
  const valor = payload.payment?.value ?? 0;
  const ciclo = payload.subscription?.cycle;
  if (valor >= 197 || (ciclo === 'YEARLY' && valor >= 97)) return 'gestao';
  return 'entrada';
}
```
