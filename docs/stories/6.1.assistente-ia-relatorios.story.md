## Story

**As a** gestor de compliance autenticado no dashboard (plano Gestão ou Enterprise),
**I want** um assistente de IA contextual no detalhe do caso e uma tela de relatórios funcional com geração automática mensal,
**so that** eu possa obter orientação de compliance instantânea sobre cada caso e exportar relatórios executivos aprovados em PDF, sem precisar investigar a legislação manualmente.

## Acceptance Criteria

1. **Assistente contextual no detalhe do caso**
   - Painel lateral `AIAssistantPanel` no `/app/casos/[caseId]` conectado à Firebase Function `/assistant-chat`
   - A Function recebe apenas metadados do caso (`categoria`, `urgencia`, `lei_aplicavel`, `dias_em_aberto`, `status`) — nunca o texto bruto do relato
   - Respostas chegam via streaming SSE, exibidas token a token no painel
   - Indicador "digitando..." aparece antes do primeiro token
   - Disponível apenas para planos Gestão e Enterprise (verificação no server)
   - Verificação de autenticação e acesso ao case antes de processar

2. **Acesso ao relato bruto (opt-in)**
   - Se o gestor clicar "Conceder acesso ao relato completo", exibir alerta de confirmação modal
   - Ao confirmar: gravar `audit_log` com `acao: 'ai_full_access_granted'`, `case_id`, `user_id`
   - Incluir texto do relato no contexto do Claude apenas nessa sessão (não persistir)

3. **Painel de insights proativos na Visão Geral**
   - `AIInsightsCard` no `/app` (home do dashboard) exibe insights gerados por IA
   - Firebase Scheduled Function (diária, 07:00 BRT) analisa casos abertos da org e gera insights
   - Insights persistidos em `orgs/{org_id}/ai_insights` (array de strings, timestamp)
   - Disponível apenas para planos Gestão e Enterprise

4. **Tela de Relatórios funcional (`/app/relatorios`)**
   - Lista relatórios existentes com colunas: Período, Status (rascunho/aprovado/exportado), Data de geração
   - Botão "Gerar relatório do mês" aciona Function de geração sob demanda
   - Relatório exibe texto gerado por Claude antes de aprovar
   - Botões: "Aprovar e exportar como PDF" e "Solicitar revisão" (volta para rascunho)
   - Exportação PDF com: cabeçalho (logo da org, período, data), métricas, texto executivo, rodapé Portal Sigilo
   - Relatório personalizado com filtros de categoria/área/período — apenas Gestão e Enterprise
   - Histórico de relatórios (rascunho/aprovado/exportado) listado e ordenado por data

5. **Firebase Scheduled Function — geração mensal automática**
   - Executa dia 1 de cada mês às 06:00 BRT
   - Para cada org Gestão ou Enterprise: agrega dados do mês anterior, chama Claude, grava em `reports/{reportId}` com `status: 'rascunho'`
   - Envia e-mail para o admin com link de revisão
   - NUNCA inclui conteúdo de relatos individuais — apenas métricas agregadas

6. **Segurança e audit logs**
   - Toda ação no fluxo de relatórios gera `audit_log`: geração, aprovação, exportação, concessão de acesso ao relato
   - `ANTHROPIC_API_KEY` nunca exposta ao browser — toda chamada via Firebase Function
   - Verificação de plano no server antes de qualquer chamada à API Anthropic

### Streaming SSE no client

```typescript
// No componente React — consumir SSE da Firebase Function
const response = await fetch("/api/assistant-proxy", {
  method: "POST",
  headers: { "Content-Type": "application/json", Authorization: `Bearer ${token}` },
  body: JSON.stringify({ caseId, messages }),
});
const reader = response.body?.getReader();
// ler chunks e exibir tokens progressivamente
```

> Nota: Firebase Functions não suportam `EventSource` diretamente do browser para chamadas autenticadas — usar `fetch` com `ReadableStream` é mais robusto.
