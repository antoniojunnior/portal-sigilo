## Story

**As a** admin logado,
**I want** ver "Gerenciar Assinatura" (se já sou cliente Asaas) ou "Fazer Upgrade" (se ainda não sou),
**so that** o CTA na seção "Plano e Faturamento" em `/configuracoes` reflita corretamente meu estado de assinatura.

## Acceptance Criteria

### AC1 — API `/api/billing/info` consultada na página /configuracoes

- `configuracoes/page.tsx` chama `GET /api/billing/info` ao montar (somente se `user.role === "admin"`).
- Dados carregados em state local: `{ has_asaas_customer: boolean }`.

### AC2 — CTA condicional

- `has_asaas_customer === true` → botão label "Gerenciar Assinatura", `href="/app/configuracoes/faturamento"`.
- `has_asaas_customer === false` → botão label "Fazer Upgrade", `href="/app/planos"`.
- Durante carregamento → Skeleton de 80x32px no lugar do botão.

### AC3 — "Ver histórico de faturas" removido

- O `<button>` com "Ver histórico de faturas" (atualmente sem função) deve ser removido.
- Será substituído por link real em story 9.9.

### AC4 — Sem regressão

- Demais seções de `/configuracoes` (Organização, Usuários, Zona de Perigo) não são afetadas.

### AC5 — Loading e erro padrão

- Skeleton de 80x32px durante fetch de `/api/billing/info` (cobre AC2).
- Erro de rede ou HTTP ≥ 500 → silencioso: exibe botão "Fazer Upgrade" (`has_asaas_customer: false` como fallback seguro).
