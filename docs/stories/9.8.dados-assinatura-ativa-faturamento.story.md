## Story

**As a** admin logado com assinatura ativa,
**I want** ver os dados reais da minha assinatura em `/configuracoes/faturamento`,
**so that** eu tenha visibilidade sobre plano, valor, ciclo e próximo vencimento sem precisar acessar o Asaas diretamente.

## Acceptance Criteria

### AC1 — Novo Route Handler `GET /api/billing/subscription`

- Verifica sessão e `role === "admin"`.
- Lê `asaas_customer_id` da org no Firestore.
- Se ausente → retorna `{ source: "firestore", plano_ativo, data_renovacao }` com status 200.
- Se presente → chama `GET ${ASAAS_BASE_URL}/v3/subscriptions?customer={id}&status=ACTIVE&limit=1`.
- Retorna objeto normalizado:

```ts
interface SubscriptionData {
  source: "asaas" | "firestore";
  plano_ativo: string;        // "gestao" | "entrada" | "enterprise"
  valor: number | null;       // ex: 227.00
  ciclo: "MONTHLY" | "YEARLY" | null;
  proximo_vencimento: string | null; // ISO date
  status: "ACTIVE" | "INACTIVE" | "SUSPENDED" | null;
  subscription_id: string | null;   // id Asaas para uso em outras stories
}
```

### AC2 — UI: seção "Assinatura Ativa" em `/configuracoes/faturamento`

- Remove o bloco "Gerenciar assinatura" que chamava o portal externo (`handleManageSubscription` + `/api/billing/portal`).
- Substitui por card informativo com os campos de `SubscriptionData`.
- Campos exibidos:
  - Plano (label traduzido: Entrada / Gestão / Enterprise)
  - Ciclo (Mensal / Anual)
  - Valor (formatado em BRL)
  - Próximo vencimento (formatado `dd/MMM/yyyy`)
  - Badge de status (ATIVO / SUSPENSO / INATIVO)
- `source === "firestore"` → exibe aviso sutil: "Dados locais — assinatura Asaas não vinculada".
