## Story

**As a** admin logado de uma organização,
**I want** acessar `/alterar-plano` e trocar meu plano atual para upgrade ou downgrade,
**so that** eu possa ajustar o plano da minha org sem sair da plataforma, com visual familiar ao `/planos`, e toda troca seja registrada no audit log.

## Acceptance Criteria

### AC1 — Middleware protege `/alterar-plano`

- `src/middleware.ts` atualizado: `/alterar-plano` adicionado ao `config.matcher`

```ts
export const config = {
  matcher: ["/app/:path*", "/alterar-plano"],
};
```

- Acesso sem cookie → redirect `/app/login`
- Role !== "admin" → redirect `/app`

### AC2 — Server Component `page.tsx` com auth guard

- `src/app/alterar-plano/page.tsx` — Server Component (sem `"use client"`)
- Lê cookie `__session` via `cookies()` do Next.js
- Chama `verifySession(sessionCookie)` de `@/lib/utils/auth`
- Se sem cookie ou sessão inválida → `redirect("/app/login")`
- Se `session.role !== "admin"` → `redirect("/app")`
- Extrai `planoAtual: Plano` e `orgName: string` da sessão
- Renderiza `<AlterarPlanoView planoAtual={planoAtual} orgName={orgName} />`
- `data-portal` no `<main>` (light-forced, igual a `/planos`)

### AC3 — Client Component `AlterarPlanoView`

- `src/app/alterar-plano/AlterarPlanoView.tsx` — Client Component (`"use client"`)
- Recebe `{ planoAtual: Plano, orgName: string }` do Server Component
- Estado: `const [ciclo, setCiclo] = useState<BillingCycle>("anual")` — default anual
- Layout:
  - Link "← Voltar para Faturamento" → `/app/configuracoes/faturamento`
  - Contexto da org: `{orgName}`
  - H1: `"Escolha seu próximo plano"` com token `text-[var(--text-3xl)] font-bold`
  - Subtítulo mencionando `PLANO_LABELS[planoAtual]` por nome
  - `<BillingToggle>` reutilizado de `src/app/planos/BillingToggle.tsx`
  - Grid de `<PlanoCardAuth>` para cada plano com `planoAtual` passado
  - Rodapé de instrução sobre mudanças e link para faturamento
- **Proibido:** uso de `cn()` — usar `[...].filter(Boolean).join(" ")`

### AC4 — Componente `PlanoCardAuth`

- `src/app/alterar-plano/PlanoCardAuth.tsx` criado
- Interface:
  ```tsx
  interface PlanoCardAuthProps {
    plano: PlanoConfig;
    ciclo: BillingCycle;
    planoAtual: Plano;
  }
  ```
- Derivação de `ActionType`:
  ```ts
  type ActionType = "current" | "upgrade" | "downgrade" | "enterprise";
  ```
  - `enterprise` → se `plano.id === "enterprise"`
  - `current` → se `plano.id === planoAtual`
  - `upgrade` → se `ordem[plano.id] > ordem[planoAtual]`
  - `downgrade` → se `ordem[plano.id] < ordem[planoAtual]`
- Variantes visuais por `ActionType`:

| Tipo | Visual |
|---|---|
| `current` | `border-2 border-[var(--color-text-tertiary)] bg-[var(--color-bg-tertiary)]` — sem `opacity-80` (emenda A-2) |
| `upgrade` + destaque | `border-2 border-[var(--color-primary)] bg-[var(--color-primary-surface)]` |
| `upgrade` sem destaque | `border border-[var(--color-border)] bg-[var(--color-card)]` |
| `downgrade` | `border border-[var(--color-border)] bg-[var(--color-card)]` |
| `enterprise` | `border border-[var(--color-border)] bg-[var(--color-card)]` |

- Badge topo "Plano atual" em `current` — `bg-[var(--color-bg-tertiary)] text-[var(--color-text-tertiary)]`
- Bloco "Você vai desbloquear" em `upgrade`: `bg-[var(--color-primary-surface)]` (emenda A-3 — sem `/5`) + lista de até 4 features novas com ícone `<Sparkles>`
- Aviso de downgrade em `downgrade`: `role="alert"`, `bg-[var(--color-warning-surface)]`, `text-[var(--color-warning)]`
- CTAs por tipo e ciclo:

| Tipo | Ciclo | CTA | Variant |
|---|---|---|---|
| `current` | — | "Plano atual" disabled | `ghost disabled` |
| `upgrade` | mensal | "Fazer upgrade" | `primary` |
| `upgrade` | anual | "Fazer upgrade — Anual" | `primary` |
| `downgrade` | mensal | "Fazer downgrade" | `secondary` |
| `downgrade` | anual | "Fazer downgrade — Anual" | `secondary` |
| `enterprise` | — | "Falar com vendas" | `secondary` |

- Botão `current` com `aria-label="Plano atual ativo"` e `disabled`
- Botão upgrade/downgrade com `aria-label="Fazer upgrade para {nome do plano}"` / `"Fazer downgrade para {nome do plano}"`
- **Proibido:** `cn()`, `opacity-80`, `bg-[var(--color-success)]/10`

### AC5 — API `POST /api/checkout/change-plan`

- `src/app/api/checkout/change-plan/route.ts` criado
- Auth obrigatória:
  - Lê cookie `__session`
  - `verifySession()` → 401 se ausente/inválida
  - `role !== "admin"` → 403
- Body: `{ plano: "entrada" | "gestao", ciclo: "mensal" | "anual" }`
- Validações:
  - Plano e ciclo obrigatórios e válidos → 400 se inválidos
  - `plano === session.plano` → 400 `"Plano já ativo"`
- Usa `createPaymentLink(plano, ciclo)` de `src/lib/asaas/createPaymentLink.ts` (criado na Story 9.5)
- Retorna `{ url: string }`
- **Registro no audit_log (imutável):**
  - Documento em `audit_logs/{orgId}/events` com campos:
    - `tipo: "troca_plano"`
    - `plano_anterior: string`
    - `plano_novo: string`
    - `ciclo: string`
    - `uid: string`
    - `criado_em: Timestamp`
  - Nenhum update ou delete sobre esse documento (Firestore Rules já bloqueiam)

### AC6 — Botão "Alterar plano" em `/faturamento`

- `src/app/(dashboard)/app/(protected)/configuracoes/faturamento/page.tsx` recebe botão:
  ```tsx
  <Link href="/alterar-plano">
    <Button variant="secondary" size="sm" className="gap-2">
      <ArrowUpDown size={14} />
      Alterar plano
    </Button>
  </Link>
  ```
- Posicionado na seção de plano atual, após o badge do plano
