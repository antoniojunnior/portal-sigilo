Plano de correção · Org suspensa/cancelada cria gestores ilimitados

BUG-20260721-R4T8 (#2) · contexto unificacao-plano-assinatura · gerado 2026-07-21
criticalP0

Defeito e causa raiz

firestore.rules#getPlanoLimit retorna null ("sem limite") para suspenso/cancelado — sentinela herdado de quando null significava "plano Enterprise, sem limite real". A Rule de create em users interpreta null como liberado, sem checar contagem. Confirmado contra emulador Firestore real: teste de org suspensa criando usuário deveria falhar e não falhou (antes do fix).

Estratégia

Correção direta. Trocar o sentinela null por 0 pros dois estados bloqueados. 0 naturalmente barra qualquer criação via getOrgUsersCount(...) < getPlanoLimit(...) (contagem nunca é negativa), sem precisar reescrever a lógica da Rule consumidora.

// atual
return plano == 'suspenso' || plano == 'cancelado' ? null : 50;

// corrigido
return plano == 'suspenso' || plano == 'cancelado' ? 0 : 50;
