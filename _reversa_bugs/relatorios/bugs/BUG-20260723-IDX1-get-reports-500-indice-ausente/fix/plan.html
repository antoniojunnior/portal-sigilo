Defeito

GET /api/reports/generate retornava 500 Internal Server Error em produção, derrubando toda a página /app/relatorios sem workaround (o botão manual "Gerar relatório" foi removido pela própria feature 005).

Causa raiz (confirmada via evidência de código)

O commit 03f61f7 (deploy de índices + feature 005) nunca teve, no firestore.indexes.json, uma entrada para collectionGroup: "reports" com os campos que o GET usa (org_id + gerado_em) — confirmado por git show 03f61f7 -- firestore.indexes.json. Se um índice equivalente existia só no console Firebase (nunca capturado no arquivo local), o firebase deploy --only firestore:indexes dessa mesma sessão o teria removido (o CLI trata o arquivo local como fonte única de verdade). Sem try/catch, a exceção FAILED_PRECONDITION do Firestore subia crua como 500 genérico.

O código já foi corrigido no commit 73241bb (mesma sessão, ~15min depois do incidente): índice adicionado + try/catch no GET. Já está em origin/main há ~9h sem novos relatos de 500.

Estratégia escolhida

Correção direta — sem debate, causa única sem hipótese concorrente, risco baixo. Como o código já está corrigido e ao vivo, o trabalho restante é documentar corretamente o bug (que nunca completou o ciclo) e blindar contra recorrência com um teste de regressão estrutural.
